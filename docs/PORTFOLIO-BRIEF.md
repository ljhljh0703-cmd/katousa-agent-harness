# Portfolio Brief — 카투사

Status: evidence-backed draft. Claims are limited to verified implementation and fixed-fixture results.

## 한 문장

초보 투자자에게 맞춰 설명 방식은 성장하고, 출처·위험 고지·비권유·비실행 원칙은 고정하는 AI 투자 이해 파트너.

## 핵심 카피

> 설명은 나에게 맞게. 안전 기준은 누구에게나 같게.

보조 카피:

> 수익률은 못 올려도, 이해도는 올려드립니다.

## 문제 정의

초보 투자자의 “지금 사도 될까요?”는 단순한 정보 질문이 아니다. 투자 기간, 손실 감수 범위, 현재 정보의 출처가 비어 있는 상태에서 결론을 요구하는 질문이다. 카투사는 결론을 더 그럴듯하게 만드는 대신, 빠진 정보와 위험, 판단이 달라지는 조건을 드러낸다.

개인화도 통제 대상이다. 설명 선호는 기억할 수 있지만, 한 번의 행동이나 손익을 보고 투자 목표와 위험성향까지 추정해서는 안 된다.

## 직접 내린 설계 결정

1. 질문 해석과 설명 생성은 언어 모델에 맡겼다.
2. 출처·금지 표현·상태 변경·반복 중단은 코드가 검사하게 했다.
3. 설명 형식은 한 번에 한 필드만 변경 후보로 만든다.
4. 투자 목표·기간·손실 감수 범위는 사용자 확인 없이는 바꾸지 않는다.
5. 손익 결과는 사용자 성향과 안전 규칙의 변경 근거로 쓰지 않는다.

## 역할 분리

| 역할 | 책임 | 구현 근거 |
|---|---|---|
| AI 에이전트 | 질문 분류, 설명 작성, 이해 확인 질문 | `SKILL.md` |
| 하네스 | 출처·금지 표현·출력 형식·상태 변경 검사 | `contracts.py`, `validate_output.py` |
| 성장 루프 | 사건 기록, 한 필드 후보, 사용자 확인, 버전 적용, 반복 중단 | `state.py`, `replay.py` |

## 포트폴리오 서사

1. 초보 투자자가 결론부터 묻는 장면으로 시작한다.
2. 답변 품질보다 통제 가능한 개인화가 문제라고 정의한다.
3. 모델이 맡은 일과 코드가 막는 일을 보여 준다.
4. 이벤트 ID 재사용, 실행 순서 경쟁, 로그 오탐을 어떻게 고쳤는지 설명한다.
5. 29개 테스트와 30회 반복 실행 결과를 제시한다.
6. 실제 사용자 검증 전에는 이해도 향상을 주장하지 않는다고 끝낸다.

## 공개 증거

- 단위 테스트: `29`개 통과
- 반복 실행: `30`회, `15 PASS / 15 expected FAIL`
- 설명 형식: checklist·example·numbers
- 안전 기준 불일치: `0`건
- 공개 파일 검사: 로컬 절대경로·이메일·비밀 문자열 `0`건
- GitHub Actions: Python 3.11·3.12 통과
- 증거 스냅샷: `docs/portfolio-evidence.json`
- 주장 원장: `docs/EVIDENCE-MAP.md`

## 한계

- 투자 자문·종목 추천·주문 실행을 제공하지 않는다.
