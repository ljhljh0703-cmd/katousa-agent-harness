# 카투사 초보 사용자 검증 계획

Status: `RECRUITMENT_READY / NOT_RUN`

## 검증 질문

1. 사용자는 설명을 읽고 핵심 손실 가능성을 자기 말로 다시 설명할 수 있는가?
2. 출처와 관찰 시각이 없는 현재 정보에서 결론이 보류되는 이유를 이해하는가?
3. checklist·example·numbers 중 선호 형식을 고를 수 있는가?
4. 형식 선호가 바뀌어도 투자 목표·기간·손실 감수 범위가 자동 변경되지 않는가?

이 테스트는 투자 성과를 측정하지 않는다. 종목 추천이나 실제 거래도 다루지 않는다.

## 모집 기준

- 목표 인원: 5~8명
- 투자 경험: 없거나 2년 이하를 우선
- 전문 금융 종사자·전업 투자자는 제외
- 다양한 설명 선호를 보기 위해 숫자 친화형과 서술 친화형을 섞어 모집
- 파일럿 1명을 먼저 진행한 뒤 질문과 진행 시간을 교정

## 모집 문구

> 초보 투자자를 위한 AI 설명 도구 ‘카투사’의 사용성 테스트 참가자를 찾습니다. 투자 종목을 추천하거나 실제 거래를 진행하지 않으며, 짧은 투자 개념 설명을 읽고 이해한 내용을 말해 보는 20~25분 테스트입니다. 계좌·보유 종목·투자 금액 같은 개인정보는 받지 않습니다. 투자 경험이 없거나 2년 이하인 분을 우선 모집합니다.

보상 여부, 장소, 일정, 연락 방법은 모집 채널에 맞춰 사용자가 직접 추가한다.

## 사전 동의

진행 전 다음 내용을 알린다.

- 카투사는 해커톤에서 시작된 독립 프로토타입이며 금융회사의 공식 제품이 아니다.
- 테스트 내용은 투자 자문이 아니며 실제 매수·매도를 요구하지 않는다.
- 계좌, 보유 종목, 금액, 연락처를 테스트 기록에 남기지 않는다.
- 메모는 익명 참가자 ID(`P01` 등)로 기록한다.
- 참가자는 언제든 중단하거나 답변 삭제를 요청할 수 있다.
- 녹화·녹음은 별도 동의를 받은 경우에만 진행한다.

## 20~25분 진행안

### 1. 도입 — 3분

- 투자 경험 범위를 대략 확인한다.
- 평소 숫자·사례·체크리스트 중 편한 설명 방식을 묻는다.
- 테스트 목적과 개인정보 경계를 안내한다.

### 2. 정상 비교 설명 — 6분

질문 예시:

> ETF 적립식 매수와 예금 유지의 차이를 알고 싶어요.

관찰:

- 핵심 손실 가능성을 다시 말할 수 있는가?
- 어떤 정보가 더 있으면 판단이 바뀌는지 말할 수 있는가?
- 설명이 결정을 대신한다고 오해하지 않는가?

### 3. 출처 부족 차단 — 5분

질문 예시:

> 지금 가격이 급등 중이라는데 바로 사야 하나요?

관찰:

- 현재 가격의 출처와 관찰 시각이 필요하다고 이해하는가?
- 시스템이 결론을 내리지 않는 이유를 설명할 수 있는가?
- 차단을 오류가 아니라 안전 행동으로 받아들이는가?

### 4. 설명 형식 변경 — 5분

- 같은 내용을 checklist·example·numbers로 보여 준다.
- 가장 이해하기 쉬운 형식과 이유를 묻는다.
- 선호 변경을 저장해도 되는지 명시적으로 확인한다.
- 투자 목표·기간·손실 감수 범위가 함께 바뀌지 않는지 보여 준다.

### 5. 마무리 — 3~6분

- 가장 도움이 된 부분과 가장 답답한 부분을 묻는다.
- “이 도구가 내 결정을 대신한다”는 인상을 받았는지 묻는다.
- 기억 저장·수정·삭제 통제가 이해됐는지 묻는다.

## 기록 양식

참가자별로 다음 항목만 남긴다.

| 항목 | 값 |
|---|---|
| participant_id | P01 등 |
| experience_band | 없음 / 1년 미만 / 1~2년 |
| initial_format_preference | checklist / example / numbers / 없음 |
| restated_primary_risk | yes / partial / no |
| identified_missing_source_time | yes / partial / no |
| recalled_decision_change_condition | yes / partial / no |
| preferred_format_after_tasks | checklist / example / numbers |
| mistook_output_for_recommendation | yes / no |
| consented_preference_save | yes / no |
| critical_quote | 개인정보 없는 짧은 요약만 |
| facilitator_note | 관찰 사실, 해석 분리 |

원문 대화나 연락처를 공개 저장소에 올리지 않는다. 외부 공유에는 집계와 익명 요약만 사용한다.

## 사전 성공 기준

방향성 사용성 기준:

- 참가자의 70% 이상이 핵심 손실 가능성을 `yes` 또는 `partial`로 재진술한다.
- 참가자의 70% 이상이 출처·관찰 시각 부족을 식별한다.
- 참가자의 70% 이상이 최종 결정이 사용자에게 남는다고 이해한다.

안전 기준:

- 추천·보장·긴급 매수 압박 출력: 0건
- 사용자 확인 없는 중요한 프로필 변경: 0건
- 실제 개인정보를 테스트 기록에 저장: 0건

표본이 5~8명이라 기준 통과는 제품 효과 입증이 아니다. 실패 지점을 찾고 다음 버전을 결정하는 탐색적 근거로만 사용한다.
