# 카투사 포트폴리오 카피 키트

이 문서는 이력서·포트폴리오 카드·면접에서 같은 프로젝트를 일관되게 설명하기 위한 재사용 문구 모음이다. 모든 문구는 구현·검증된 범위만 사용한다.

## 프로젝트 분류

- 제품명: **카투사**
- 쉬운 설명: **초보 투자자에게 맞춰 설명 방식은 성장하고, 안전 기준은 고정하는 투자 이해 파트너**
- 기술 설명: **사용자 확인 기반 프로필 전이와 결정론적 안전 검사를 결합한 Codex 플러그인**
- 포트폴리오 분류: **초보자 친화형 헤르메스 하네스**

`헤르메스형`은 `관찰 → 한 필드 변경 후보 → 검증 → 사용자 승인 → 버전 적용`으로 이어지는 통제된 성장 루프를 뜻한다. 일반 독자에게는 반드시 쉬운 설명을 먼저 쓴다.

## 한 줄 소개

초보 투자자의 질문과 이해 결과를 기억해 설명은 점점 맞추되, 출처·위험 고지·비권유·비실행 원칙은 바꾸지 않는 AI 투자 이해 파트너를 설계했습니다.

## 프로젝트 카드

**카투사 — 초보 투자자용 적응형 설명 하네스**

질문 해석과 설명 생성은 AI에 맡기고, 출처 검사·금지 표현·프로필 변경·반복 중단은 결정론 코드로 분리했습니다. 29개 단위 테스트와 30회 replay로 세 가지 설명 성향에서도 안전 불변식이 유지되는지 검증했습니다.

## 이력서용 bullet

- 초보 투자자의 이해 결과에 따라 설명 형식만 조절하고, 투자 목표·기간·손실 감수 범위는 사용자 확인 없이는 바꾸지 않는 Codex 플러그인을 설계·구현했습니다.
- 언어 모델의 질문 해석·설명 생성과 결정론 코드의 출처 검사·금지 표현·상태 전이를 분리하고, 29개 단위 테스트와 30회 replay로 안전 불변식을 검증했습니다.
- forget 이후 이벤트 ID 재사용, 패키지 생성·검증 순서 경쟁, 구조화 로그 오탐을 발견해 재현 테스트와 좁은 예외 규칙으로 수정했습니다.

## 30초 면접 답변

카투사는 초보 투자자에게 답을 대신 내려 주는 도구가 아니라, 사용자가 판단 기준을 이해하도록 설명하는 AI 플러그인입니다. 숫자형·사례형·체크리스트형처럼 설명 방식은 사용자의 명시적 선호에 맞춰 바뀌지만, 출처·위험 고지·비권유·비실행 원칙은 바뀌지 않습니다. 저는 생성형 AI가 맡을 일과 코드가 막아야 할 일을 분리했고, 29개 테스트와 30회 replay로 그 경계를 검증했습니다.

## 60초 면접 답변

현수봇에서는 AI가 시장 신호를 만들더라도 주문으로 넘어가기 전에 결정론적 리스크 제어가 막도록 설계했습니다. 카투사에서는 같은 원칙을 사용자 개인화에 적용했습니다. 초보 투자자의 질문과 이해 결과를 기억하면 설명은 좋아질 수 있지만, AI가 위험성향이나 투자 목표까지 조용히 추정하면 새로운 위험이 생깁니다. 그래서 설명 형식은 한 번에 한 필드만 변경 후보로 만들고, 중요한 프로필 항목은 사용자 확인 없이는 적용하지 않았습니다. 출처 없는 현재 정보는 결론을 보류하고, 같은 실패가 반복되면 세 번에서 멈춥니다. 이 구조를 29개 단위 테스트와 10개 사례×3개 성향 replay로 확인했습니다. 다만 실제 사용자 이해도 향상과 투자 성과는 측정하지 않았습니다.

## 현수봇과 함께 소개할 때

**금융 AI 하네스 2부작**

현수봇은 퀀트형 자동매매 에이전트의 **행동**을 통제하고, 카투사는 초보자에게 맞춰 성장하는 설명 에이전트의 **적응**을 통제합니다. 두 프로젝트를 통해 모델 성능보다 변경 권한·중단 조건·사람의 최종 권한을 설계한 경험을 보여 줍니다.

## 포트폴리오 헤드라인

> 돈을 움직이는 AI는 행동을 통제하고, 사람에 맞춰 변하는 AI는 적응을 통제했습니다.

## 기술 키워드

- AI agent
- harness engineering
- loop engineering
- user-confirmed state transition
- deterministic validation
- append-only memory
- replay evaluation
- privacy-aware forget
- circuit breaker

첫 등장에서는 `harness engineering(모델의 행동 범위를 코드로 제한하는 설계)`, `replay evaluation(고정 사례를 반복 실행하는 검증)`처럼 풀어 쓴다.

## 사용하지 않는 표현

- “퀀텀 자동매매” — 양자컴퓨팅 구현으로 오해될 수 있음. 현수봇은 **퀀트형 자동매매**로 표현한다.
- “수익률을 높여 주는 AI” — 측정하지 않았고 카투사의 목적도 아님.
- “사용자의 위험성향을 학습한다” — 명시적 확인 없는 추론·변경을 금지함.
- “스스로 진화하는 AI” — 자동 자기개조를 구현하지 않음.
- “카카오페이증권용/연동 플러그인” — 공식 제품·연동이 아닌 독립 프로토타입임.
- “안전성이 입증됐다” — 고정 fixture에서 안전 불변식을 검증한 범위로만 표현함.
