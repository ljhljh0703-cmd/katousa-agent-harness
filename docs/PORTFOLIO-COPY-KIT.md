# 카투사 포트폴리오 카피 키트

이 문서는 이력서·포트폴리오 카드·면접에서 카투사를 같은 방향으로 설명하기 위한 재사용 문구 모음이다. 모든 문구는 구현하고 검증한 범위만 사용한다.

## 프로젝트 분류

- 제품명: **카투사**
- 쉬운 설명: **초보 투자자에게 맞춰 설명 방식은 성장하고, 안전 기준은 고정하는 투자 이해 파트너**
- 기술 설명: **사용자 확인 기반 상태 변경과 결정론적 안전 검사를 결합한 Codex 플러그인**
- 설계 분류: **초보자 친화형 적응 하네스**

하네스는 AI가 해도 되는 일과 하면 안 되는 일을 코드로 나눈 안전장치다. `관찰 → 변경 후보 → 검증 → 사용자 확인 → 버전 적용` 순서로만 설명 방식을 바꾼다.

## 포트폴리오 헤드라인

> 설명은 나에게 맞게. 안전 기준은 누구에게나 같게.

## 한 줄 소개

초보 투자자의 질문과 이해 결과를 기억해 설명은 점점 맞추되, 출처·위험 고지·비권유·비실행 원칙은 바꾸지 않는 AI 투자 이해 파트너를 설계했습니다.

## 프로젝트 카드

**카투사 — 초보 투자자용 적응형 설명 하네스**

질문 해석과 설명 생성은 AI에 맡기고, 출처 검사·금지 표현·프로필 변경·반복 중단은 결정론 코드로 분리했습니다. 29개 단위 테스트와 30회 반복 실행으로 세 가지 설명 형식에서도 안전 판단이 달라지지 않는지 확인했습니다.

## 이력서용 bullet

- 초보 투자자의 이해 결과에 따라 설명 형식만 조절하고, 투자 목표·기간·손실 감수 범위는 사용자 확인 없이는 바꾸지 않는 Codex 플러그인을 설계·구현했습니다.
- 언어 모델의 질문 해석·설명 생성과 결정론 코드의 출처 검사·금지 표현·상태 변경을 분리하고, 29개 단위 테스트와 30회 반복 실행으로 안전 조건을 검증했습니다.
- forget 이후 이벤트 ID 재사용, 의존 작업의 실행 순서 경쟁, 구조화 로그 오탐을 발견해 재현 테스트와 좁은 예외 규칙으로 수정했습니다.

## 30초 면접 답변

카투사는 초보 투자자에게 답을 대신 내려 주는 도구가 아니라, 사용자가 판단 기준을 이해하도록 설명하는 AI 플러그인입니다. 숫자형·사례형·체크리스트형처럼 설명 방식은 사용자의 명시적 선호에 맞춰 바뀌지만, 출처·위험 고지·비권유·비실행 원칙은 바뀌지 않습니다. 저는 생성형 AI가 맡을 일과 코드가 막아야 할 일을 분리했고, 29개 테스트와 30회 반복 실행으로 그 경계를 확인했습니다.

## 60초 면접 답변

초보 투자자는 “지금 사도 될까요?”처럼 결론부터 묻지만, 투자 기간·손실 감수 범위·현재 정보의 출처가 비어 있으면 좋은 문장만으로 안전한 설명이 되지 않습니다. 카투사는 이 문제를 답변 생성이 아니라 통제 가능한 개인화 문제로 풀었습니다. 사용자가 숫자형 설명을 선호한다는 사실은 기억하지만, 행동이나 손익을 보고 위험성향까지 추정하지 않습니다. 설명 형식은 한 번에 하나만 변경 후보로 만들고, 중요한 프로필 항목은 사용자 확인 없이는 적용하지 않습니다. 출처가 없으면 결론을 보류하고, 같은 실패가 세 번 반복되면 멈춥니다. 이 구조를 29개 단위 테스트와 10개 사례×3개 설명 형식으로 확인했습니다. 실제 사용자 이해도 향상과 투자 성과는 아직 측정하지 않았습니다.

## 기술 키워드

- AI agent
- harness engineering
- loop engineering
- user-confirmed state transition
- deterministic validation
- append-only memory
- replay evaluation
- privacy-aware forget
- circuit breaker

첫 등장에서는 `하네스 엔지니어링(AI의 행동 범위를 코드로 제한하는 설계)`, `반복 실행 평가(고정 사례를 여러 설명 형식으로 다시 돌리는 검증)`처럼 풀어 쓴다.

## 사용하지 않는 표현

- “수익률을 높여 주는 AI” — 측정하지 않았고 카투사의 목적도 아님.
- “사용자의 위험성향을 학습한다” — 명시적 확인 없는 추론·변경을 금지함.
- “스스로 진화하는 AI” — 자동 자기개조를 구현하지 않음.
- “카카오페이증권용/연동 플러그인” — 공식 제품·연동이 아닌 독립 프로토타입임.
- “안전성이 입증됐다” — 고정 사례에서 안전 조건을 검증한 범위로만 표현함.
